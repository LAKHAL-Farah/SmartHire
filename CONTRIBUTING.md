Thank you for your interest in contributing to SmartHire.

This repository is maintained with a focus on engineering quality, reproducible
local development, and secure handling of secrets. Please follow these
guidelines when contributing:

- Open an issue to discuss significant changes before implementing them.
- Keep commits small, focused, and well-documented.
- Do not commit secrets or credentials. Use `.env.example` as a template.
- Run existing linters and tests locally before opening a PR.
- Follow the code style used by the module you modify.

For maintainers: add specific CI checks, branch protections, and release notes
as appropriate.
